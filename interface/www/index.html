What is git?

How to install and run git

Git is a program you have to run on your computer. The installation steps depend on your operating system.
  • Windows
  • macOS / OS X
  • Linux

First, let's check if git is already installed. Open the command-promt (press start and type cmd) or git bash (press start and type git bash). Then type the following and press enter. git --version If you get something similar to git version 2.19.0 it is installed! Read on how to configure your name further down.

Installing

Download git here and run the installer. Try the above mentioned test to see if the install was sucsessfull.

Creating a user

Now with git installed we need to make certain git knows who we are, see if it already does by typing git config user.name If it responds with a name you've got everything set up. Otherwise type
git config --global user.name "Your Name"
git config --global user.email "your@email.com"

GitHub is a website were git repositories can be stored, just as here on git.dhack.se. It is the largest service of it's kind, but in no way affiliated with git the software.

However since the tools hosted by GitHub are very usefull we recommend creating an account for yourself there. GitHub also has a free student license. To create your account go to github.com/join to and sign up with your chalmers email. When you have an account the student plan is enabled by visiting education.github.com/pack and klicking "Get your pack" and follow the instructions on the page.

This excercise is meant for two or more people to co-operate, find yourself a partner and follow the instructions. Also make sure you have created a GitHub account, if you haven't the excercise "GitHub Account" has instructions on how yo do that.

Now we're gonna try to co-operate as if we had been working on lab together. The instructions will not be very specific and you are more free to do what you like, but if you get stuck, don't hesitate to ask questions!

To make things easier we recommend one of you host the project on their GitHub. Decide whom of you it will be. That person will have to create a new repository on GitHub. Tick the "Initialize this repository with a README" checkbox to make cloning the repository a little bit easier.

Now, to allow others to push to your repository, go to project-settings by pressing the tab "Settings" with a little gear icon. Then go to Collaborators and add the others. They will get a notification on GitHub and an email with an invite link.

Now all of you clone the project and play around a bit. Try pushing and pulling as much as you can. Can you identify when you get a conflict and when you don't?

Well, what is git then?

Git is a powerful tool for version control and cooperative coding. But to many it seems very complicated and hard to wrap your head around. This leads people to avoid using it properly and in turn gives them even more trouble.

This site was part of a workshop on git at Chalmers where we tried to introduce people to git in a not so frightening way. The presentation can be found here.
